# 思政智题云枢

这是一个基于 `PyQt6` 的桌面应用，用来整理思政选择题题库。当前项目以“本地 SQLite 总库 + 开始页菜单导入/导出”为核心流程，支持文档导题、库内查重、AI 补解析，以及 Markdown / xlsx 导入导出。

## 当前功能

- 开始页题库浏览：
  - 左侧按 `必修三 -> 第x课 -> 第x框` 展示库内层级
  - 右侧表格展示题目、选项、答案、解析
  - 支持双击条目打开对话框并修改常用字段
  - 支持视图菜单控制显示哪些列，且列显示状态会持久化
- 导入弹窗中的确认导入资料页：
  - 使用统一表格显示当前已选 `docx`、版本、层级、图片检查、表格检查与操作列
  - 支持选择/拖拽补充文档
  - 支持通过“打开并编辑”打开 `doc` 目录中的备份 `Word` 文档进行调整
  - 关闭已编辑的 `Word` 后，会自动刷新图片/表格检查结果
- AI 导入 `docx`
- AI 导入详情页：
  - 题号与题型解析页会展示当前项目配置中的题号题型模型返回结果与一致性结论
  - 对比结论不一致时，“题型”列留空并要求用户手动选择
  - 题目内容解析页会展示当前项目配置中的题目内容模型耗时、JSON 结果与一致性结论
  - 支持红黄高亮区分少数派结果与空返回
  - 支持 `Ctrl + 滚轮` 缩放题目内容解析表
- 确认导入资料页可逐文档填写各自的 `版本 / 层级`
- 多文档导入时，确认资料页先统一处理；点击“开始导题”后，会按文档拆成多个独立导入窗口，并从“题号与题型解析”页分别继续
- 程序设定支持控制导入流程是否默认执行库内查重
- 程序设定支持控制导入文档时是否自动生成解析（`自动生成 / 不生成`）
- 程序设定支持控制导入完成时是否显示本次 `docx` 解析费用
- 导入流程不再保留单独的“查重选项页 / 解析选项页”，是否进入后续步骤由程序设定直接决定
- 库内查重：直接对当前总库 `.db` 做相似题比对，无需指定路径
- AI 生成解析：对缺失解析的题目批量生成解析并写回题库
- Markdown 导出：从开始页菜单栏导出当前选中层级的 Markdown
- xlsx 导入导出：
  - 支持从数据库字段表 `xlsx` 直接导入
  - 支持导出当前层级为 `xlsx`
  - 支持导出整体数据库表为 `xlsx`

## 并发设置

- 程序设定中可设置默认题库保存位置，默认值为 `桌面/思政题库`
- 程序设定中的“导入设定”可设置默认导入目录：
  - `下载`
  - `桌面`
  - `自定义路径`
- 当自定义导入目录不存在时，导入文件对话框会自动回退到 `下载`
- API 配置中的“项目配置”页提供两个独立并发数：
  - `题目内容解析并发数`
  - `答案解析生成并发数`
- 两个并发数范围均为 `1 ~ 8`
- 两个并发数默认值均为 `3`

## 模型配置

- 在“开始”页的“设置 -> 配置API”中维护模型配置
- API 配置页包含：
  - `项目配置`
  - `DeepSeek`
  - `Kimi`
  - `千问`
- 各厂商页提供“可用模型列表”，支持查看、增加、修改模型
- 千问预设模型列表包含 `qwen3.6-plus`
- 项目配置中统一维护：
  - `题号题型解析` 模型与轮次/比例
  - `题目内容解析` 模型与轮次/比例
  - `解析生成模型`
  - 两个并发数
- 其中 `题号题型解析` 的轮次固定为 `1`
- API 配置底部使用一个“统一测试”按钮，位于“确定”左边
- “统一测试”会：
  - 刷新三家余额
  - 对本次有变动的厂商 API 做连通性测试
  - 对项目配置中的模型做联测
- AI 生成解析阶段仍可选择 `DeepSeek / Kimi / 千问`
- AI 生成解析默认模型为 `DeepSeek / deepseek-reasoner`
- 当选择 `DeepSeek` 时，解析生成使用其单独的解析模型配置
- 配置采用 OpenAI 兼容接口形式：`base_url / api_key / model`

详细说明见 [AI接口与提示词](docs/AI接口与提示词.md)。

## 运行方式

1. 安装依赖

```bash
pip install -r requirements.txt
```

1. 启动应用

```bash
python main.py
```

## 使用流程

1. 在开始页确认 API 配置，并按需在“项目配置”中调整解析轮次、比例和两个并发数
2. 浏览当前总库 `.db` 中的题目，必要时可直接双击编辑
3. 在开始页点击“导入 -> 从文档文件解析导入”，打开独立导入弹窗
4. 在弹窗中确认导入资料（支持拖拽补充文件），逐文档填写 `版本 / 层级`，查看图片/表格检查结果；必要时可通过“打开并编辑”打开 `doc` 目录中的备份 Word 调整，关闭后界面会自动刷新
5. 点击“开始导题”后，如当前选择多个文档，会按文档拆成多个独立导入窗口，并从“题号与题型解析”页分别开始处理
6. 在“题号与题型解析”页查看当前题号题型模型识别结果与一致性结论；对比结论不一致时，需手动选择“题型”后才能继续
7. 在“题目内容解析”页查看逐题返回结果、耗时与一致性结论
8. 按程序设定决定是否执行库内查重
9. 按程序设定决定是否自动执行 AI 补解析
10. 导入弹窗完成后自动写入当前题库 `.db`，并可在完成页查看本次 `docx` 解析费用总和与按厂商拆分的费用明细表
11. 回到开始页继续浏览，或在菜单栏导出 Markdown / xlsx

更完整的页面流程见 [向导流程](docs/向导流程.md)。

## 当前页面

- 主窗口当前保留 `登录` 与 `开始` 两页；资料导入使用独立导入向导弹窗。
- 主窗口不再内嵌导入页栈，导入相关页面仅在独立弹窗中注册与运行。
- 从开始页进入文档导入时，程序会优先复用 `题库目录/doc/` 下内容哈希一致的备份文档，避免重复复制相同 `Word` 文件。
- `登录` 页目前主要是占位入口：
  - `本地登录` 直接进入开始页
  - `用户登录` 仅弹出失败提示，不是完整账号体系
- 页面名称如下：
  - `登录`
  - `开始`
  - `确认导入资料`
  - `题号与题型解析`
  - `题目内容解析`
  - `库内查重（结果）`
  - `AI 生成解析`
  - `导入数据库成功`
- 其中部分页面按流程分支决定是否进入：
  - 从开始页菜单打开导入弹窗后，会依次进入 `确认导入资料`、`AI 解析详情`
  - 程序设定启用库内查重时会进入 `库内查重（结果）`
  - 程序设定启用自动解析生成时会进入 `AI 生成解析`

## 题库格式

- 当前题库主存储为本地 SQLite 数据库文件
- 数据库路径由“常规设定”中的默认题库保存位置决定
- 默认位置为 `桌面/思政题库/思政题库.db`
- 同时保留两类 `xlsx` 流程：
  - 标准题库表：用于传统 5 列题库读写
  - 数据库字段表：用于数据库记录导入导出
- 标准题库表头为：
  - `编号`
  - `题目`
  - `选项`
  - `答案`
  - `解析`

详细格式见 [数据结构与题库格式](docs/数据结构与题库格式.md)。
