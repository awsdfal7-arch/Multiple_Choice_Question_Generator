# 思政智题云枢

这是一个基于 `PyQt6` 的向导式桌面应用，用来整理思政选择题题库。项目以本地 `xlsx` 作为题库存储，支持 AI 导题、手动录题、跨库查重、AI 补解析和导出 Markdown。

## 当前功能

- 建题库：创建题库文件夹与 `{题库名称}.xlsx`
- 两种录题方式：
  - AI 导入 `txt / docx`
  - 手动录入题干、选项、答案、解析
- AI 导入详情页：
  - 显示当前题号、总题数、轮次、各模型耗时
  - 展示 `DeepSeek / Kimi / 千问` 三模型返回结果与一致性结论
  - 支持红黄高亮区分少数派结果与空返回
- AI 编辑与确认：导入后可逐题检查、修改、删除
- 跨库查重：对指定文件夹内的多个题库做相似题比对
- AI 生成解析：对缺失解析的题目批量生成解析并写回题库
- 导出：主向导当前支持导出 Markdown
- 中途不落盘：项目文件夹、题库 `xlsx`、导出 `md` 均在最后一步统一写入，减少用户取消时的残缺数据

## 并发设置

- 程序设定中提供统一并发数选择：`1 / 2 / 3 / 4 / 5`
- 程序设定中可设置默认题库保存位置，默认值为 `桌面/思政题库`
- 程序设定中使用一个可输入下拉框设置解析生成模型，格式为 `服务商 / 模型名`
- 解析生成模型默认值为 `DeepSeek / deepseek-reasoner`
- 解析生成模型每次修改后都需要先测试通过，才能保存程序设定
- 该设置同时作用于：
  - AI 导题时的题级并发
  - AI 生成解析时的批量并发
- 默认并发数为 `3`

## 模型配置

- 在“开始”页可打开“配置 API…”
- AI 导题阶段需要：
  - `DeepSeek`
  - `Kimi`
  - `千问`
- AI 生成解析阶段可在程序设定中选择 `DeepSeek / Kimi / 千问`
- AI 生成解析阶段的模型名支持手动输入，默认使用 `deepseek-reasoner`
- 当选择 `DeepSeek` 时，解析生成使用其单独的解析模型配置
- 配置采用 OpenAI 兼容接口形式：`base_url / api_key / model`

详细说明见 [AI接口与提示词](docs/AI接口与提示词.md)。

## 运行方式

1. 安装依赖

```bash
pip install -r requirements.txt
```

2. 启动应用

```bash
python main.py
```

## 使用流程

1. 在开始页确认 API 配置，并按需打开程序设定调整统一并发数
2. 创建题库草稿，默认保存位置为 `桌面/思政题库`
3. 选择录题方式：
   - AI 导入
   - 手动录入
4. 可选执行跨库查重
5. 可选执行 AI 补解析
6. 最后一步统一写入项目文件夹、`xlsx` 与 Markdown，并自动打开输出目录

更完整的页面流程见 [向导流程](docs/向导流程.md)。

## 当前导航页

- 当前启用的是“常规向导”导航模式；批量处理入口在开始页中仍为未实现状态。
- 常规向导当前共注册 `16` 个页面。
- 页面名称如下：
  - `登录`
  - `开始`
  - `建题库`
  - `选择录题模式`
  - `手动录入`
  - `选择资料文件`
  - `AI 解析详情`
  - `AI 编辑与确认`
  - `题目预览与编辑`
  - `查重选项`
  - `跨库查重（设置）`
  - `跨库查重（结果）`
  - `解析选项`
  - `AI 生成解析`
  - `输入名称`
  - `导出`
- 其中部分页面按流程分支决定是否进入：
  - 选择 `手动录入` 时会进入 `手动录入`
  - 选择 `AI 导入` 时会进入 `选择资料文件`、`AI 解析详情`、`AI 编辑与确认`
  - 开启跨库查重时会进入 `查重选项`、`跨库查重（设置）`、`跨库查重（结果）`
  - 开启解析生成时会进入 `解析选项`、`AI 生成解析`
  - 当题库名称为占位名时会进入 `输入名称`

## 题库格式

- 每个题库对应一个文件夹
- 题库文件名为 `{题库名称}.xlsx`
- 工作表名默认为 `questions`
- 标准表头为：
  - `编号`
  - `题目`
  - `选项`
  - `答案`
  - `解析`

详细格式见 [数据结构与题库格式](docs/数据结构与题库格式.md)。
