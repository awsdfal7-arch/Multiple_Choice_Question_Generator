# 数据结构与题库格式（xlsx）

题库文件为 .xlsx，建议使用单个工作表保存题目数据。

默认保存方式：

- 每个题库对应一个文件夹
- 文件夹内的题库文件为：`{题库名称}.xlsx`

## 工作表

- 工作表名：questions
- 第一行为表头（字段名），后续每一行为一题

## 字段（最小集）

标准题库 xlsx（工作表 questions）包含以下列（表头为中文）：

- 编号：题目编号（可由程序自动生成，也可手动填写）
- 题目：题干（必填）
- 选项：选项（必填，JSON 字符串）
- 答案：答案（必填）
- 解析：解析（可为空）

说明：

- 选项采用 JSON 字符串表示，便于支持可变选项数与保留原始标识
  - 例：`{"A":"xxx","B":"yyy","C":"zzz","D":"..."}`
- 答案采用大写字母序列表示
  - 单选：`A`
  - 多选：`ACD`

字段命名映射（用于代码内部处理）：

- 编号 -> id
- 题目 -> stem
- 选项 -> options
- 答案 -> answer
- 解析 -> analysis

## 题型与选项标识映射

资料中常见两类标识：

- 单选：A/B/C/D...
- 多选：①②③④...

题库内部统一存储为 A/B/C/D...，映射规则为“按出现顺序映射”：

- ① -> A
- ② -> B
- ③ -> C
- ④ -> D
- ⑤ -> E

## 可选字段（后续可扩展）

- id：题目唯一标识（UUID 或自增）
- source：来源（文件名/章节等）
- created_at：入库时间
- updated_at：更新时间
- tags：标签（JSON 数组字符串）
- difficulty：难度（数字或枚举）

## 校验规则（建议）

- 题目非空
- 选项至少 2 项
- 选项 key 为 A-Z 的连续字母或可映射的顺序标识
- 答案中每个字母必须在选项 keys 中存在
