# 向导流程

本项目当前采用“开始页 + 独立导入向导弹窗”交互：开始页负责浏览题库、配置、导入与导出；资料导入在单独弹窗中按页面顺序完成资料确认、AI 导题、库内查重、补解析与入库。当前主流程已经移除了“建题库 / 选择录题模式 / 手动录入 / 命名页 / 跨库查重设置页 / 查重选项页 / 解析选项页”等旧环节，页面标题以功能命名为准。

## 开始

- 显示欢迎信息与 API 配置入口。
- 可打开“配置 API…”维护 DeepSeek / Kimi / 千问配置。
- 可打开“常规设定”调整默认题库保存位置、默认导入目录、是否显示导入费用等。
- 可在 API 配置的“项目配置”中调整题号题型解析、题目内容解析、解析生成模型与两个并发数。
- 可通过菜单“导入 -> 从文档文件解析导入”打开独立导入弹窗。
- 可通过菜单“导入 -> 从表格文件直接导入”导入数据库字段表 xlsx。
- 可通过菜单“导出 -> 导出当前层级为 Markdown”导出当前选中层级。
- 可通过菜单导出当前层级或整体数据库表为 xlsx。
- 左侧树展示当前总库层级：
  - `必修三`
  - `第一课`
  - `第一框`
- 右侧表格展示当前总库题目，支持：
  - 视图菜单切换显示列
  - 双击条目编辑
  - 只读浏览题目、选项、答案、解析

## 导入弹窗

- 当前不再出现单独的“查重选项”与“解析选项”页面。
- AI 解析完成后，是否进入“库内查重（结果）”与“AI 生成解析”由程序设定直接决定。

## 确认导入资料

- 页面使用统一表格显示当前已选文件、版本、层级、图片检查、表格检查与操作列。
- 支持选择或拖拽 `docx`。
- 层级要求填写为三级数字格式，例如 `3.2.2`。
- 打开文件对话框时，默认目录来自“导入设定”；默认值为 `下载`。
- 若用户配置的自定义目录不存在，则自动回退到 `下载`。
- 会检查 `Word` 文档中是否存在图片、是否存在表格。
- 支持通过“打开并编辑”打开 `doc` 目录中的备份 `Word`。
- 用户保存并关闭 `Word` 后，图片/表格检查结果会自动刷新。
- 导入前会优先按文件内容哈希复用 `doc` 目录中已存在的相同文档，避免重复复制。
- 点击“开始导题”后自动开始解析。
- 若当前选择多个文档，会在此时按文档拆成多个独立导入窗口，并从“题号与题型解析”页分别继续。

## 题号与题型解析

- 页面主体只保留结果表格与底部按钮。
- 以表格展示当前资料在题号题型解析模型组下的识别结果。
- 底部左侧提供“打开文档”按钮，用于打开当前选中行对应的源文档。
- 当对比结论不一致时，“题型”列留空并允许用户通过下拉框手动选择；未完成选择前不能继续。
- 具体模型、轮次与通过比例由 API 配置中的“题号题型解析”项目配置决定。
- 其中“题号题型解析”的轮次固定为 `1`。

## 题目内容解析

- 按上一页确认过的题号列表逐题解析题目内容。
- 以表格展示每题当前题目内容解析模型组的耗时、JSON 结果与一致性结论。
- 页面不再显示处理文件路径，也不再显示模型可用性标签。
- 底部左侧提供“停止解析”和“重试”按钮。
- 题目内容解析的模型、最高轮次、通过比例与并发数均由 API 配置中的“题目内容解析”项目配置决定。
- 表格支持：
  - `Ctrl + C` 复制为 `tab / 换行` 文本
  - `Ctrl + 滚轮` 缩放
  - 用户手动调整列宽

## 库内查重（结果）

- 直接对“常规设定”指定的当前题库 `.db` 执行查重。
- 无需手工指定查重路径。
- 以只读表格展示相似题对：
  - 当前题目
  - 当前序号
  - 库内来源
  - 所属层级
  - 相似度
- 双击行可打开详情对话框，并列查看当前题目与库内题目的题干、选项与答案。

## AI 生成解析

- 自动读取当前草稿中缺失解析的题目。
- 进入本页后自动开始批量生成，提供进度条与停止生成按钮。
- 生成完成或停止后，自动把解析写回当前草稿数据。
- 参考项来自程序设定与本地参考资料：
  - `reference/resource/` 下的全部 md
  - `reference/mistakes/选择题常见错题归因与答题策略分析.md`

## 导入数据库成功

- 当前批次题目会在进入本页前写入当前题库 `.db`。
- 本页显示本次实际写入题数。
- 若导入设定启用了费用显示，本页会展示本次 `docx` 解析费用总和，以及按厂商拆分的费用明细表：
  - 厂商
  - 本次调用模型
  - 原本余额
  - 当前余额
  - 本次费用
- 点击“返回开始页”后关闭弹窗，可继续浏览总库或用开始页菜单导出 Markdown / xlsx。
