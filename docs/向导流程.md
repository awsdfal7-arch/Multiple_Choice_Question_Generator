# 向导流程

本项目采用“向导式（Wizard）”交互：按页面顺序完成建库、导入/录入、查重、补解析与导出。由于存在分支（AI 导入/手动录入、是否查重、是否生成解析），页面标题不再使用固定 Step 编号，而以功能命名为准。

## 开始

- 显示欢迎信息与 DeepSeek/Kimi/千问 配置状态。
- 可选：点击“配置 DeepSeek…”保存 DeepSeek API 配置。
- 可选：点击“配置 Kimi…”保存 Kimi API 配置（用于导入阶段的校验）。
- 可选：点击“配置 千问…”保存千问 API 配置（用于导入阶段的校验）。
- 每个配置对话框提供“测试 API”按钮；需先测试通过，才能点击确定保存并生效。
- 点击“下一步”进入向导。

## 建题库

- 题库名称可留空：
  - 若留空，会先创建一个占位名称的题库，后续会自动/手动重命名。
  - 若选择 AI 导入且名称留空：选择资料文件后，会自动用资料文件名作为题库文件夹名与 xlsx 名。
  - 若选择手动录入且名称留空：在导出前会新增“输入名称”页面用于重命名。
- 选择保存位置（data 或自定义父目录）。
- 创建题库文件夹与 `{题库名称}.xlsx`。
- 题库 xlsx 标准表头：编号、题目、选项、答案、解析。

## 选择录题模式

- AI 导入：资料文件 -> AI 解析详情 -> AI 编辑与确认 -> 写入题库。
- 手动录入：逐题输入/粘贴 -> 写入题库 -> 题目预览与编辑。

## AI 导入路径

**选择资料文件**
- 选择 docx/txt（可多选）。
- 点击“下一步”后自动开始解析。

**AI 解析详情**
- 显示解析进度与停止按钮。
- 以表格展示每题三模型返回：DeepSeek/Kimi/千问返回内容、各模型耗时、轮次与一致性结论。
- 导入抽取采用三模型一致性校验：每题最多 3 轮，每轮依次由 DeepSeek/Kimi/千问各抽取 1 次；同一轮三者结果一致才写入，否则继续下一轮，三轮仍不一致则跳过该题。

**AI 编辑与确认**
- 以表格展示通过一致性校验的题目（题目/选项/答案），支持双击编辑与删除。
- 点击下一步后把当前表格内容写入题库，并进入“查重选项”。

## 手动录入路径

**手动录入**
- 逐题输入编号、题干、选项、答案、解析（除必填项外可留空）。
- 点击“追加到题库”直接写入题库。

**题目预览与编辑**
- 以表格展示当前题库的全部题目，支持编辑。
- 点击“保存到题库”覆盖写回题库文件。

## 查重选项

- 运行跨库查重：进入“跨库查重（设置）/（结果）”。
- 跳过跨库查重：直接进入“解析选项”。

## 跨库查重（设置）

- 设定要对比的文件夹路径（默认 data 文件夹）。
- 设定相似度阈值（0~1，默认 0.85）。
- 点击“下一步”后开始查重，并进入“跨库查重（结果）”。

## 跨库查重（结果）

- 以只读表格展示相似题对（左/右文件名、序号、相似度）。
- 双击行可打开详情对话框，并列查看两侧题目的题干、选项与答案。

## 解析选项

- 生成解析：进入“AI 生成解析”。
- 跳过解析生成：直接进入“导出”。
- 当选择“生成解析”时，可同时配置：
  - 自动参考 `reference/` 文件夹内所有 md（如果存在，可关闭）
  - 常见错题归因参考：`common_mistakes/选择题常见错题归因与答题策略分析.md`（如果存在）

## AI 生成解析

- 自动读取当前题库中缺失解析的题目。
- 进入本页后自动开始批量生成，提供进度条与停止生成按钮。
- 生成完成或停止后，自动把解析写回题库文件。

## 输入名称

- 仅在“建题库”时名称留空且选择“手动录入”时出现。
- 在导出前输入题库名称，用于重命名题库文件夹与 xlsx 文件名。

## 导出

- 设定 Markdown 导出路径（必选）。
- 点击向导的“导出并打开文件夹”完成导出并打开题库所在文件夹。
