# 向导流程

本项目当前采用“开始页 + 独立导入向导弹窗”交互：开始页负责浏览总库、配置和 Markdown 导出；资料导入在单独弹窗中按页面顺序完成解析、层级补充、库内查重、补解析与入库。当前主流程已经移除了“建题库 / 选择录题模式 / 手动录入 / 命名页 / 跨库查重设置页”等旧环节，页面标题以功能命名为准。

## 开始

- 显示欢迎信息与 API 配置入口。
- 可打开“配置 API…”维护 DeepSeek / Kimi / 千问配置。
- 可打开“程序设定”调整统一并发数、默认输出目录等。
- 可通过菜单“导入 -> 从文档文件解析导入”打开独立导入弹窗。
- 可通过菜单“导出 -> 导出当前层级为 Markdown”导出当前选中层级。
- 左侧树展示当前总库层级：
  - `必修三`
  - `第一课`
  - `第一框`
- 右侧表格展示当前总库题目，支持：
  - 视图菜单切换显示列
  - 双击条目编辑
  - 只读浏览题目、选项、答案、解析

## 导入弹窗

## 选择资料文件

- 页面为左右布局。
- 左侧上方：
  - 选择或拖拽 `docx / txt`
  - 显示当前已选文件
- 左侧下方：
  - 导入提醒区域
  - 会检查 `Word` 文档中是否存在图片、是否存在表格
- 右侧：
  - 文档预览区域
  - `txt` 直接显示文本内容
  - `docx` 转为 PDF 后预览
- 点击“下一步”后自动开始解析。

## AI 解析详情

- 显示解析进度与停止按钮。
- 以表格展示每题三模型返回：
  - DeepSeek
  - Kimi
  - 千问
- 同时显示各模型耗时、轮次与一致性结论。
- 导入抽取采用三模型一致性校验：
  - 每题最多 3 轮
  - 每轮依次由 DeepSeek / Kimi / 千问 各抽取 1 次
  - 同一轮三者结果一致才写入
  - 三轮仍不一致则跳过该题

## AI 编辑与确认

- 以表格展示通过一致性校验的题目。
- 支持双击编辑与删除。
- 点击下一步后进入 `level_path` 补充页。

## 补充层级归属

- 为本次文档解析得到的题目统一填写 `level_path`
- 例如：
  - `3.6.3`
- 该信息用于后续归类、入库与查重展示。

## 查重选项

- 运行库内查重：进入“库内查重（结果）”
- 跳过库内查重：直接进入“解析选项”

## 库内查重（结果）

- 直接对当前总库 `converted_db/思政题库.db` 执行查重。
- 无需手工指定查重路径。
- 以只读表格展示相似题对：
  - 当前题目
  - 当前序号
  - 库内来源
  - 所属层级
  - 相似度
- 双击行可打开详情对话框，并列查看当前题目与库内题目的题干、选项与答案。

## 解析选项

- 生成解析：进入“AI 生成解析”
- 跳过解析生成：直接进入“导入数据库成功”
- 当选择“生成解析”时，可同时配置：
  - 自动参考 `reference/resource/` 文件夹内所有 md（如果存在，可关闭）
  - 常见错题归因参考：`reference/mistakes/选择题常见错题归因与答题策略分析.md`（如果存在）

## AI 生成解析

- 自动读取当前草稿中缺失解析的题目。
- 进入本页后自动开始批量生成，提供进度条与停止生成按钮。
- 生成完成或停止后，自动把解析写回当前草稿数据。

## 导入数据库成功

- 当前批次题目会在进入本页前写入当前总库 `converted_db/思政题库.db`。
- 本页显示本次实际写入题数。
- 关闭弹窗后返回开始页，可继续浏览总库或用开始页菜单导出 Markdown。
